# V2 真实 API 基线结论（2026-07-14）

> 这是固定追问下线前的 V2.0 历史基线。V2.1 已改为宽泛问题直接作答，澄清相关数字不再作为当前门禁。

## 总结论

当前版本的完整 Release Gate 结论是 **BLOCKED**。

硬门禁中，安全、单轮单问题、最多两问和执行错误率全部通过；唯一硬门禁失败是幻觉率：**5/48 = 10.42%**，高于 5% 的上限。

## 为什么 57/62 PASS 仍然被阻塞

综合分会被检索命中、相关性和格式拉高。S24、S25、S27 和 S41 虽被 Judge 标记为存在无依据断言，综合分仍高于 80。因此，幻觉和安全不能被其他分数抵消，必须作为独立硬门禁。

这个案例非常适合面试时说明“为什么不能只评一个总分”。

## 五道 PARTIAL

- S02、S08、S10、S38：本应直接回答的概念/总览题被澄清路由拦住。
- S26：走了知识回答，但 Faithfulness 只有 1/5。

## 优化优先级

### P0：将幻觉率压到 5% 以下

1. 在生成 Prompt 中强制“具体费用、时间、门槛、机构名称和政策结论必须可在上下文逐句对齐”。
2. 片段不支持时明确说“现有信息不足”，不用模型常识补空。
3. 对 S24–S27 先改检索/query，再改 Prompt，分两次对照评测，避免不知道提升来自哪一层。
4. 为高风险具体数字加答案后的句子级 groundedness 检查。

### P1：减少过度澄清和路由误判

- 保护“X 是什么”、“有哪些常见类型”、“普通人需要哪些账户”等可直答问法。
- 补齐“转钱”等口语化资金意图。
- 对“为什么不要相信稳赚”等安全科普反例设置放行规则，降低误拒。

### P2：优化检索排序

Hit@5 已经很高，但弱标签 nDCG@5 仅 62.12%。下一步重点应是 query 改写与 rerank，而不是继续单纯扩大 Top-K。

## 面试可说的结论

> 我用 62 道真实 API 题做端到端评测，表面 PASS 率是 91.9%，安全违规为 0，但分层评测发现 48 个知识答案中有 5 个存在无依据断言，因此幻觉硬门禁仍然阻止发布。这让我能区分“召回不到”、“路由走错”和“召回到但模型自行补充”，并把优化优先级放在 grounded generation，而不是继续追求一个更高的模糊总分。
