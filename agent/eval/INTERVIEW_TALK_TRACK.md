# 简历与面试表达手册

## 简历 bullet（保守可证明版）

## 简历 bullet（结果版，数字以报告为准）

> 搭建 RAG Agent 分层自动化评测，在 52 道检索题上达到 Hit@5 100%、MRR 99.0%，在 21 个交互场景上实现 Need-Clarify F1 94.7%、单轮单问题率和最多两问合规率 100%；通过失败分析识别检索排序、口语资金意图和安全误拒等优化点。

> 完成 62 道真实 API 端到端评测，严格 PASS 率 91.9%、Faithfulness 4.50/5、Answer Relevance 5.00/5、安全违规 0，p95 时延 8.07s；通过独立幻觉硬门禁识别 5/48 个无依据生成样本，避免高综合分掩盖金融科普风险。

注：不要把当前 100% Hit@5 说成“回答准确率 100%”。它只表示弱金标下 Top-5 至少有一个相关片段。

## 90 秒面试讲法

> 我做的是一个面向美股初学者的 RAG 科普 Agent。我没有只用一个“准确率”，而是先定义用户任务成功，然后拆成路由、澄清、检索、生成、安全和系统六层。这样失败时可以判断是意图判错、没找到，还是找到了但模型胡编。
>
> 数据上，我用知识库目录乘题型做覆盖矩阵，现有 62 道核心问答，另外为“宽泛问题先问 1–2 个问题”做了 21 个多轮剧本，特别测该不该问、每轮只问一个、最多两问、中途跳过、换题和安全优先级。
>
> 工程上有两条流水线：离线流水线不调模型，几秒内跑检索、路由和多轮状态机；端到端流水线调真实模型，用 LLM-as-Judge 评忠实度和相关性。每次报告会记录评测集、知识库和 Prompt 指纹，并跑 Release Gate。
>
> 第一轮基线也给出了很具体的产品结论：召回很强，但弱标签 nDCG 暴露了排序还可以优化；澄清的单问题和最多两问已经稳定，但口语化“转钱”和安全边界反例还会判错。这些结果可以直接变成下一轮产品和算法优先级。

## 面试官追问与回答

### 为什么不只看总分？

总分无法归因。Hit 低要修检索；Hit 高但 Faithfulness 低要修 Prompt 或模型；如果是路由错，后面的检索和生成再好也无意义。安全还不能被其他高分抵消，所以要单独做硬门禁。

### 为什么同时看 Precision、Recall、MRR 和 nDCG？

Hit@5 只说明找到过一个。Precision 看噪声，Aspect Recall 看多要点是否完整，MRR 看第一个相关结果是否靠前，nDCG 看高相关度结果的整体排序。这几个指标分别回答不同问题。

### LLM-as-Judge 可信吗？

它适合大规模快速回归，但不是真理裁判。我会固定 temperature，限制 Judge 只根据当次上下文打分，再用 20% 人工双标样本算一致性、±1 分一致率和 MAE。生成模型与 Judge 相同时还要主动承认自偏好风险。

### 为什么宽泛问题只问 1–2 个？

追问可以提高针对性，但每多一轮都会增加放弃风险。所以我把它设计成有停止条件的状态机：只问会明显改变答案路径的信息，每轮一个，最多两个，允许用户跳过或换题。上线后用 Helpful Rate 和 Clarification Abandonment 做 A/B 验证。
