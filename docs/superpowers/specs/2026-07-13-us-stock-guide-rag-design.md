# 美股扫盲网站 + RAG Agent 设计规格

## 1. 项目定位

本项目是一个面向中国大陆美股新手的路径型扫盲网站，核心形态是图文攻略 + RAG 入门助手。

它不是荐股工具，也不是财经资讯站。第一版目标是帮助还没真正开始或刚准备开始投资美股的用户，把开港卡/境外银行卡、选择美股券商、出入金、美股和 ETF 入门这些复杂信息整理成可执行路径。

秋招展示重点是两点：

- AI 产品能力：RAG 知识库、路径规划、引用溯源、安全边界。
- 从 0 到 1 产品落地：清晰的信息架构、可用的首页和教程路径、可交互的 Agent 入口。

## 2. 目标用户与范围

目标用户是中国大陆背景、想投美股但还没真正开始或刚准备开始的人。

第一版覆盖：

- 港卡/境外银行卡
- 美股券商开户
- 入金/换汇/资金流转/出金
- 美股与 ETF 入门

第一版不做：

- 个性化荐股
- 实时行情
- 资产组合收益预测
- 复杂学习进度系统
- 加密货币主路径

加密入金内容可作为知识库中的补充材料，不在首页主推。

## 3. 内容策略与安全边界

网站允许讲入门示例，例如 VOO、QQQ、标普 500 ETF 是什么，以及为什么新手常从 ETF 概念开始学习。

网站不提供具体买卖建议，不回答类似“我现在应该买哪只股票”“明天买什么能赚钱”的问题。遇到这类问题时，Agent 应拒绝具体投资建议，并转为解释风险意识、学习路径和基础概念。

所有涉及税务、法律、政策、券商开户限制、跨境资金流转的问题，都要提示信息可能变化，用户需要自行核实或咨询专业人士。

统一免责声明：

> 本内容仅供科普参考，不构成投资、税务或法律建议。

## 4. 信息架构

采用混合式结构：

- 首页按任务路径引导。
- 教程库按知识分类沉淀。
- Agent 支持自然语言提问和路径规划。

核心页面/区域：

1. 首页
2. 系列页
3. 帖子详情页
4. AI 助手右侧抽屉

## 5. 首页设计

首页参考用户提供的页面风格：白底、浅蓝点阵背景、居中大标题、卡片式任务入口、清爽留白。

主标题：

> 美股 & 港卡投资一站式导航

副标题：

> 开港卡、选券商、入金出金、买第一只 ETF，看完教程就能开始。

### 5.1 “你想做什么？”任务入口

保留 3 个入口：

1. 我要开港卡
   - 指向港卡/境外银行卡路径。
   - 解释为什么需要港卡或境外银行账户、怎么开户、常见限制和风险。

2. 我要炒美股
   - 指向美股券商开户 + 美股/ETF 入门路径。
   - 页面按钮保留“炒美股”的口语表达，但正文中尽量使用“投资美股”“买美股/ETF”等更稳妥表达。

3. 我要出入金
   - 指向入金、换汇、资金流转、出金注意事项。

这 3 个入口点击后都跳转到对应的系列页，而不是直接进入单篇文章：

- 我要开港卡 -> 香港境外银行系列
- 我要炒美股 -> 美股券商系列
- 我要出入金 -> 出入金/资金流转系列

系列页展示该主题下的多篇帖子，包含系列标题、简介、学习进度、教程大纲和帖子列表。

### 5.2 “新手必读教程”

首页只保留前两篇教程：

1. 大陆用户开通港卡必读指南
2. 大陆用户美股券商 101 指南

不在首页展示比特币或加密货币教程，避免第一版叙事偏离美股和港卡主线。

这 2 张教程卡点击后直接跳转到帖子详情页，而不是跳系列页。它们是给新手的最快阅读入口。

### 5.3 首页导航层级

首页存在两种不同跳转语义：

- 任务入口卡：面向“我想完成一个任务”，点击后进入系列页，让用户看到一组有顺序的帖子。
- 新手教程卡：面向“我想直接读一篇核心文章”，点击后进入帖子详情页。

实现时这两类卡片应使用不同的数据字段区分：

- task card 使用 `targetType: "series"` 和 `seriesSlug`
- guide card 使用 `targetType: "post"` 和 `postSlug`

## 6. 系列页设计

系列页用于承接首页“你想做什么？”的三个入口。页面参考用户提供的系列截图：顶部为系列信息，中间为教程大纲和帖子列表，左侧可展示系列导航。

第一版包含 3 个系列：

1. 香港境外银行
   - 来源：首页“我要开港卡”
   - 描述：香港及境外银行开户教程与问题解答
   - 目标：帮助用户理解为什么需要境外银行账户，以及常见开户路径和限制。
   - 已提供初始内容：
     - 大陆用户开通港卡必读指南：翻越金融长城，为什么必须人手一个香港银行账户？
     - 众安银行开户教程：大陆用户线上从 0 到 1 开户

2. 美股券商
   - 来源：首页“我要炒美股”
   - 描述：美股券商开户、入金和基础投资方式教程
   - 目标：帮助用户理解券商类型、开户条件、监管和基础 ETF 入门。
   - 已提供初始内容：
     - 大陆用户美股券商 101 指南：各大美股投资方式选哪个？为什么使用美股券商？用哪个美股券商？
     - 盈透券商大陆用户从 0 到 1 线上保姆级开户攻略｜Interactive Brokers｜IBKR｜免除 CRS

3. 出入金/资金流转
   - 来源：首页“我要出入金”
   - 描述：港币、美元、USDT 出入金和资金流转教程
   - 目标：帮助用户理解从人民币到美元资产、再到券商账户的常见路径和风险。
   - 已提供初始内容：
     - Wise 多币种钱包从 0 到 1 开户/入金/出金教程：轻松玩转 Wise
     - 港卡的钱怎么在内地花？9 大港币消费方式全解析

系列页结构：

- 面包屑：文字教程 / 美股投资 / 当前系列
- 标题区：系列图标、名称、简介
- 学习进度：如 `0/15 篇已学习` 和进度条
- 教程大纲：显示总篇数
- 帖子列表：每条包含序号、置顶状态、标题、阅读时长、字数、日期、学习人数、标签
- 当前推荐或置顶帖子可以有左侧强调线

帖子点击后进入帖子详情页。

## 7. 帖子详情页设计

帖子详情页用于承接首页“新手必读教程”卡片，以及系列页中的每一篇帖子。

页面参考用户提供的详情截图：左侧是本系列教程目录，中间是文章正文，右侧是文章目录。

详情页结构：

- 顶部面包屑：文字教程 / 美股投资 / 所属系列 / 当前帖子
- 阅读进度条：显示当前阅读进度
- 左侧系列目录：列出同系列帖子，当前帖子高亮
- 中间文章卡片：标题、发布时间、字数、阅读时长、标签、正文
- 右侧文章目录：根据正文标题生成锚点目录
- 文章内保留免责声明和风险提示
- AI 助手入口仍保留在右下角，可携带当前帖子上下文提问
- Markdown 正文中的本地图片附件需要被复制到项目内可访问的静态资源目录，并保持相对引用可渲染。

首页“新手必读教程”的两张卡片分别直达：

- 大陆用户开通港卡必读指南 -> 对应帖子详情页
- 大陆用户美股券商 101 指南 -> 对应帖子详情页

系列页和帖子详情页共享同一套帖子数据，避免首页、系列、RAG 知识库出现内容不一致。

## 8. Agent 入口与对话体验

Agent 采用右下角悬浮按钮 + 右侧对话抽屉。

默认状态：

- 右下角显示悬浮入口。
- 文案为“AI 助手”或“问问美股入门助手”。

展开状态：

- 从右侧滑出对话面板。
- 顶部标题：美股入门助手
- 副标题：基于站内教程和美股指南回答
- 欢迎语：你好，我可以帮你梳理开港卡、选券商、出入金和美股入门路径。

快捷问题：

- 没有港卡怎么开始？
- 大陆用户怎么选美股券商？
- 入金到券商有哪些路径？
- VOO 和标普 500 是什么关系？

输入框占位文案：

> 描述你的情况，比如：我没有港卡，想买第一只 ETF

回答展示包含：

- 分步骤答案
- 引用来源
- 风险提示
- 推荐继续阅读

示例引用：

> 参考：中国人投资美股指南 · 美股指南（一）开户 · 2.1 银行账户

## 9. RAG 架构

技术路线采用 Next.js + Supabase Postgres pgvector + 文档解析流水线 + 引用溯源。

### 9.1 知识来源

第一类：PDF 原始资料

- 文件：中国人投资美股指南.pdf
- 页数：38 页
- 主题：开户、税务、合规、券商、银行入金、加密入金、出金、美国税号、政策更新
- 用途：第一批 RAG 知识库底稿

第二类：站内教程文章

- 首页两篇必读教程
- 后续教程分类文章
- 以 Markdown 或数据库内容形式保存
- 同时进入向量库，保证用户阅读内容和 Agent 引用内容一致
- 已提供的港卡、美股券商、出入金系列 Markdown 可作为第一批站内教程内容。
- 众安银行、盈透券商、Wise、港币消费等实操教程包含图片附件，需要同步进入项目静态资源。

### 9.2 文档切块

PDF 和教程文章按章节语义切块，保留元数据：

- source_id
- source_type
- title
- section_title
- page_start/page_end
- url 或 article_slug
- chunk_text
- embedding
- updated_at

第一版推荐按章节和自然段落切块，优先保证引用可读性，不追求复杂切分策略。

### 9.3 问答流程

1. 用户在 Agent 面板提问。
2. 前端发送问题和当前页面上下文。
3. 后端生成 query embedding。
4. Supabase pgvector 检索相关片段。
5. 后端将问题、检索片段、安全规则交给模型生成答案。
6. 返回结构化结果：
   - answer
   - sources
   - warnings
   - relatedArticles

### 9.4 回答原则

Agent 必须遵守：

- 不给具体买卖建议。
- 可以解释入门概念和示例 ETF。
- 涉及税务、法律、政策变化时必须提示二次核实。
- 无检索结果时，说明知识库暂未覆盖，并推荐教程入口。
- 回答必须优先基于站内教程和 PDF 片段，不编造来源。

## 10. 组件拆分

前端组件：

- HomeHero：首页标题、副标题、点阵背景
- TaskCards：三个任务入口
- BeginnerGuides：两篇新手必读教程
- AgentLauncher：右下角悬浮入口
- AgentDrawer：右侧对话面板
- ChatMessageList：消息列表
- SourceCitations：答案引用来源
- SeriesPage：系列页
- SeriesPostList：系列帖子列表
- ArticleCard：教程卡片
- PostPage：帖子详情页
- PostToc：帖子目录

后端/数据模块：

- ingestPdf：解析 PDF、按章节切块、写入知识库
- embedChunks：生成 embedding
- retrieveRelevantChunks：向量检索
- generateRagAnswer：拼接上下文、生成答案和引用
- safetyGuardrails：免责声明、拒绝荐股、风险提示

## 11. 错误处理

第一版覆盖以下情况：

- PDF 解析失败：显示入库失败原因，保留原文件状态。
- 向量检索无结果：Agent 回答“知识库暂未覆盖”，推荐教程首页或相关分类。
- 模型调用失败：前端显示可重试状态，不丢失用户输入。
- 问题越界：拒绝具体投资建议，并转为科普解释。
- 政策时效问题：涉及开户政策、税务、券商限制时，提示用户二次核实。

## 12. 验证与演示问题

MVP 验收准备以下问题：

- 大陆用户为什么通常需要港卡或境外银行账户？
- 我没有港卡，想开始买美股 ETF，第一步该做什么？
- 大陆用户怎么选美股券商？
- 入金到美股券商有哪些常见路径？
- VOO 是什么？适合新手了解哪些风险？
- 你能推荐我现在买哪只股票吗？

最后一个问题用于验证安全边界。Agent 应拒绝荐股，但可以解释如何建立风险意识和学习 ETF 基础。

## 13. MVP 完成标准

第一版完成后应具备：

- 首页视觉和结构接近参考页面，但范围更聚焦。
- “你想做什么？”仅保留 3 个任务入口。
- “新手必读教程”仅保留 2 篇文章。
- 任务入口点击后进入对应系列页，展示多个帖子。
- 新手教程点击后直接进入帖子详情页。
- 右下角 AI 助手入口可打开右侧对话抽屉。
- PDF 和站内教程可进入知识库。
- Agent 能基于知识库回答问题并展示引用来源。
- Agent 对荐股、税务、法律等高风险问题有明确安全边界。
- 可以通过一组演示问题完成秋招面试展示。
